## Runtime

Use Bun for all commands (`bun`, `bunx`, `bun run`). Bun auto-loads `.env`.

## Commands

```sh
bun run build            # tsdown -> dist/
bun run test             # vitest run (all tests)
bun run test:watch       # vitest watch
bun run typecheck        # tsc --noEmit
bun run lint             # oxlint
bun run lint:fix         # oxlint --fix
bun run format           # oxfmt
bun run format:check     # oxfmt --check
bun run check            # lint + format:check
bun run generate:fonts   # regenerate src/errors/fonts.json
bun run generate:metrics # regenerate src/metrics/data.json
```

Run single test file: `bun run test -- tests/unit/css/font-face.test.ts`

Configs live in `.configs/` (tsdown, vitest, oxlint, oxfmt, commitlint). `@` alias -> `src/` in tests.

## Architecture

This is a Vite plugin (`@pyyupsk/vite-fonts`) that downloads and self-hosts web fonts at build time.

**Entry points** (built by tsdown):

- `src/index.ts` — main `fonts()` plugin export (`"."`)
- `src/frameworks/astro.ts` — Astro integration wrapper (`"./astro"`)
- `src/middleware.ts` — standalone middleware (`"./astro/middleware"`)
- `src/cli/index.ts` — `vite-fonts` CLI (clean/list)
- `src/client.d.ts` — ambient types for `@pyyupsk/fonts` virtual module (`"./client"`)

**Plugin lifecycle** (`src/hooks/`):

1. `config-resolved` — normalize user config, set up paths
2. `build-start` — download font files via `src/cache/` + `src/sources/`
3. `resolve-id` / `load` — serve virtual module `@pyyupsk/fonts` (CSS + type defs)
4. `transform-html` — inject `<link>` tags into HTML
5. `generate-bundle` / `close-bundle` — copy font assets, patch output HTML

**Key data flow**:

- User config (`FontsInput`) -> `src/config/normalize.ts` -> `NormalizedFamily[]`
- `NormalizedFamily[]` + source -> `src/sources/google.ts` -> download URLs
- Downloaded files cached in `node_modules/.cache/vite-fonts/`
- CSS generated by `src/css/generate.ts` (font-face + fallback metrics)
- Fallback font metrics from `src/metrics/data.json` (pre-generated via fontkit)

**Virtual module** `@pyyupsk/fonts` resolves to generated CSS; `@pyyupsk/fonts/meta` resolves to JSON metadata for preload link generation.

**Tests**: unit in `tests/unit/` mirroring `src/` structure; integration in `tests/integration/`; smoke in `tests/smoke/`. Fixtures in `tests/fixtures/`.
